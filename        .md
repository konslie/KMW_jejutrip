## 2. 기술 스택 (Tech Stack)
* **Front-end:** HTML5, CSS3, Vanilla JavaScript (빌드 도구 없는 경량화 구조)
* **Map API:** Kakao Maps API & Kakao Places API (제주 지역 POI 데이터 최적화)
* **Library:** `SortableJS` (Drag & Drop 기반 일정 순서 변경)

## 3. 핵심 개발 원칙 및 아키텍처 (Architecture Guidelines)

### 3.1. 안티 할루시네이션 (정확도 100% 지향)
* **문제:** 정적 위경도 하드코딩 시 장소 변경에 유연하지 못하며, 잘못된 좌표가 렌더링될 위험 존재.
* **해결:** Kakao Places API 기반의 **동적 검색 로직** 채택. `searchName`을 렌더링 시점에 실시간으로 검색하여 좌표 반환.
* **검색 스코핑(Scoping):** 타 지역 동명 상호 검색을 원천 차단하기 위해 쿼리 문자열에 강제로 `" 제주"` 키워드 병합.

### 3.2. 모바일 퍼스트 UX & 직관적 UI
* **Sticky & Shrink Map:** 스크롤 시 상단 지도가 화면에서 사라지지 않고 높이만 50% 축소(350px ➔ 175px)되며 상단 고정(`position: sticky`).
* **일차별 독립 토글 (A/B Test):** 전체 일정을 통째로 바꾸는 대신, 각 일차별 제목 옆에 A안/B안 토글 버튼을 배치하여 현장 상황에 맞게 유연한 코스 조합 가능.
* **직관적 시각화:** 1~4일차의 텍스트, 시간 입력창, 지도 위 동선(Polyline), 출발점 커스텀 마커 라벨의 색상을 일치시켜 시각적 피로도 감소.

### 3.3. 동적 상태 관리 (State Management)
* **데이터 구조:** `common`(공통 고정 일정)과 `A`/`B`(선택 옵션) 객체를 분리하여 모듈화.
* **데이터 보존 로직 (Data Retention):** A/B 옵션 탭 전환 시, 사용자가 현장에서 직접 추가한 일정(`desc: "사용자 추가 일정"`)이 초기화되지 않고 새 배열에 병합되도록 상태 보존 처리.

## 4. 버전별 주요 작업 히스토리 (Version History)

* **v1.0:** 기본 마크업 및 정적 일정표 구현, GitHub Pages 연동 테스트.
* **v2.0:** Kakao Maps API 연동, 하드코딩 좌표 기반 동선(Polyline) 렌더링.
* **v2.1 ~ v2.4 (구조 개편):** * 정적 좌표 ➔ **Places API 실시간 검색 방식**으로 전면 리팩토링.
  * `SortableJS` 도입으로 일정 Drag & Drop 기능 추가.
  * 비동기 데이터 렌더링 시 지도 로드 에러 방지 처리 (Try-Catch 및 Fallback UI 적용).
* **v2.5 ~ v2.6 (시각화 고도화):** * 맵 필터 추가: 특정 일차 동선만 지도에 분리 표시.
  * **Auto Bounds:** 필터 선택 시 해당 동선이 화면에 꽉 차도록 자동 줌인/줌아웃 알고리즘 적용.
  * 일차별 출발 지점에 말풍선 형태의 커스텀 오버레이('n일차 출발') 도입.
* **v2.7 ~ v2.8 (현장 활용성 극대화):** * 글로벌 옵션을 **일차별 개별 A/B 옵션**으로 하향 배치.
  * 대안이 존재하는 일정에 `(대안 존재)` 붉은색 배지 자동 렌더링.
  * 검색창 엔터키 트리거 및 검색 결과 즉시 초기화 버튼 추가.

## 5. 데이터 입력 가이드 (For Future Updates)

새로운 일정을 데이터 객체(`rawData`)에 추가할 때는 아래 규격을 따릅니다.

* `name`: 리스트에 표시될 타이틀 (예: `"제주공항 (블루렌트카)"`)
* `searchName`: 카카오맵 API가 100% 찾아낼 수 있는 정확한 상호명/주소 (예: `"제주국제공항 블루렌트카"`)
* `time`: 노출될 기본 시간 (예: `"14:00"`)
* `desc`: 상세 설명이나 메뉴 (예: `"도착 및 렌터카 인수"`)
* `tips`: 주의사항이나 꿀팁. 문구 안에 `⚠️` 기호 포함 시 붉은색 경고 알림창으로 디자인 자동 변경.
